# MPDB — MITRE Parser Databases

**MPDB** (**MITRE_PARSER_DATABASES**) — комплексный инструмент для парсинга, обработки и анализа баз данных MITRE (CAPEC, CWE, CVE, ATT&CK) с удобным веб-интерфейсом.

## 📋 Возможности

### 🔹 Парсинг данных
- **CAPEC** - шаблоны атак
- **CWE** - типы уязвимостей
- **CVE** - известные уязвимости
- **MITRE ATT&CK** - тактики и техники противников

### 🔹 Обработка данных
- **Связывание** - автоматическое установление перекрестных ссылок между базами
- **Автозаполнение** - заполнение пустых полей: восстановление связей, извлечение фактов из описаний, detection из ATT&CK STIX, шаблонные фразы
- **Перевод** - быстрый многопоточный перевод выбранных полей на русский язык (Google/Yandex Translate) с дедупликацией и кэшированием
- **Нормализация** - очистка и стандартизация данных
- **AI обогащение** - 🚧 находится в разработке (заполнение полей с помощью ИИ Ollama/OpenAI)

### 🔹 Графический интерфейс
- **Панель управления** - запуск и мониторинг всех процессов в реальном времени, быстрый доступ к PDF-отчёту и локальной папке `output/`
- **Редактор баз данных** - просмотр, поиск и редактирование записей
- **Связи** - интерактивный граф перекрёстных связей записи (глубина 1-3 шага) с детальной карточкой записи по клику и переходом в редактор, а также поиск пути между двумя записями через все 4 базы
- **Редактор кэша переводов** - пагинация, поиск, сворачивание длинных текстов
- **Настройки** - конфигурация лимитов парсинга и параметров перевода

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка (опционально)

Отредактируйте `src/config.py` для настройки:
- Источников данных
- Параметров перевода (потоки, задержка)
- Шаблонов автозаполнения (`AUTOFILL_TEMPLATES`)
- Ограничений на количество записей

### 3. Запуск

#### Вариант А: Графический интерфейс (рекомендуется)

```bash
python gui/run_gui.py
```

Затем откройте браузер: **http://localhost:5000**

#### Вариант Б: Консольное меню

```bash
python src/menu.py
```

#### Вариант В: Прямой запуск всех этапов

```bash
python src/run_all.py
```

## 🎯 Подробное описание

### Этапы обработки

1. **Парсинг (Parsing)**
   - Скачивание актуальных данных из официальных источников MITRE
   - Разбор XML/JSON форматов
   - Извлечение ключевых полей
   - Сохранение в JSON

2. **Связывание (Cross-Linking)**
   - Анализ перекрестных ссылок между базами
   - Заполнение полей related_capec, related_cwe, related_mitre
   - CAPEC → ATT&CK извлекается напрямую из `Taxonomy_Mappings` (Taxonomy_Name="ATTACK") при парсинге, остальные связи дозаполняются на этом этапе
   - Добавление мер защиты (mitigations)

3. **Автозаполнение (Autofill)** — ярусная стратегия, от точного к шаблонному:
   - **Ярус A**: восстановление связей по реальным ID (реверс и транзитивность, например ATT&CK → CAPEC → CWE)
   - **Ярус B**: извлечение фактов из описаний — порты, сервисы, тип атаки, категория CWE, severity из CVSS
   - **Ярус C**: detection для ATT&CK из STIX-первоисточника (detection strategies → analytics) с переводом
   - **Ярус D**: шаблонные фразы (настраиваются в `Config.AUTOFILL_TEMPLATES`) для оставшихся пустых текстовых полей
   - Поля-связи (related_*) шаблонами не заполняются — только реальные ID

4. **Перевод (Translation)**
   - Многопоточный перевод выбранных полей на русский язык (15 потоков по умолчанию)
   - Глобальная дедупликация строк и постоянный кэш — повторные запуски почти мгновенны
   - Поддержка Google Translate и Yandex Translate

5. **AI обогащение (AI Enrichment)** — 🚧 в разработке
   - Заполнение полей с помощью ИИ (Ollama/OpenAI)
   - Изменение параметров временно недоступно

### Графический интерфейс

Веб-интерфейс предоставляет:

- **Дэшборд** с карточками статистики, лентой обновлений MITRE, управлением процессами, скачиванием PDF-отчёта и кнопкой быстрого перехода в локальную папку `output/` с JSON-базами
- **Редактор баз данных** с поиском, фильтрацией и пагинацией
- **Страницу "Связи"** с графом перекрёстных связей записи (с настраиваемой глубиной 1-3 шага и детальной карточкой записи прямо под графом) и поиском пути между двумя записями через CAPEC/CWE/CVE/ATT&CK — подробнее см. [docs/features/link-graph.md](docs/features/link-graph.md)
- **Редактор кэша переводов** с пагинацией, поиском и сворачиванием длинных текстов
- **Страницу настроек** для конфигурации лимитов и перевода
- **Real-time логи** с цветовым кодированием событий

### Консольное меню

Альтернативный интерфейс для управления процессами:

```
1. Скачать и распарсить базы (без перевода)
2. Связать данные (заполнить перекрёстные ссылки)
3. Автозаполнение пустых полей (связи/факты/STIX/шаблоны)
4. AI-обогащение (заполнить пустые поля)
5. Перевести выбранные поля
6. Запустить все этапы последовательно
7. Выбрать сервис перевода
8. Очистить кэш перевода
9. Очистить некачественные переводы из кэша
10. Настроить параметры перевода
0. Выход
```

## ⚙️ Конфигурация

Основные настройки в `src/config.py`:

```python
# Перевод
ENABLE_TRANSLATION = False          # Включить перевод при парсинге
TRANSLATE_TO = "ru"                 # Целевой язык
TRANSLATION_DELAY = 0.15            # Задержка перед каждым запросом в потоке (сек)
TRANSLATION_WORKERS = 15            # Число параллельных потоков перевода
TRANSLATION_MAX_RETRIES = 5         # Число повторных попыток при ошибке
TRANSLATION_SERVICE = "google"      # google или yandex

# Автозаполнение — шаблонные фразы для пустых полей
AUTOFILL_TEMPLATES = { ... }        # см. config.py

# AI (в разработке)
AI_PROVIDER = "ollama"              # ollama или openai
AI_MODEL = "qwen2.5:3b"             # Модель для Ollama
AI_BASE_URL = "http://localhost:11434/v1"

# Ограничения (0 = все записи)
MAX_CAPEC_RECORDS = 1000
MAX_CWE_RECORDS = 1000
MAX_CVE_RECORDS = 4000
MAX_ATTACK_RECORDS = 1000
```

## 📊 Выходные данные

После обработки в папке `output/` создаются:

- `capec_database.json` - база шаблонов атак
- `cwe_database.json` - база типов уязвимостей
- `cve_database.json` - база известных уязвимостей
- `mitre_attack.json` - база тактик и техник ATT&CK
- `translate_cache.json` - кэш переводов

Каждая запись содержит:
- ID
- Название
- Описание
- Связанные записи из других баз
- Меры защиты
- И другие поля в зависимости от типа базы

## 🆘 Решение проблем

### GUI не запускается
```bash
pip install -r gui/requirements.txt
```

### Ошибки при парсинге
- Проверьте подключение к интернету
- Увеличьте таймауты в `config.py`
- Попробуйте запустить с меньшим количеством записей

### Проблемы с переводом
- Проверьте API ключ для Yandex (если используется)
- При ошибках 429 (Too Many Requests) уменьшите `TRANSLATION_WORKERS` или увеличьте `TRANSLATION_DELAY`

### Автозаполнение не скачивает STIX
- Ярус C требует доступ к GitHub (~50 МБ). Можно указать локальный файл: `python src/step4_autofill.py --stix путь/к/enterprise-attack.json` или пропустить ярус: `--no-detection`

## 🔒 Безопасность при развёртывании

GUI предназначен для **локального использования** (на `localhost`). Перед тем как делать его доступным из сети — учтите следующее:

- **Сервер слушает все интерфейсы**: `socketio.run(app, host='0.0.0.0', port=5000, ...)` в `gui/run_gui.py` и `gui/app.py`. Это значит, что приложение доступно любому устройству в той же сети (а при проброшенном порте — и из интернета), а не только с этого компьютера.
- **Это dev-сервер Werkzeug** (`allow_unsafe_werkzeug=True`) — без защиты от нагрузки, лимитов запросов и прочих механизмов, обязательных для production.
- **Нет аутентификации**: любой, кто достучится до порта, получает полный доступ ко всем функциям без логина и пароля:
  - запуск/остановку процессов парсинга, связывания, перевода (нагрузка на CPU и сеть вашего компьютера, обращения к MITRE/NVD от его имени);
  - `PUT /api/database/<db_name>` и `PUT /api/database/<db_name>/record/<record_id>` — запись произвольного JSON в файлы `output/`. Имя базы (`db_name`) подставляется в путь к файлу без проверки по белому списку;
  - `POST /api/output-folder/open` — запускает процесс (проводник/файловый менеджер) на хост-машине по сетевому запросу.

### Рекомендации
- Для локальной работы используйте `http://localhost:5000` и не открывайте порт 5000 в роутере/файрволе наружу.
- Если нужен доступ из локальной сети (например, для лабораторной работы по безопасности), запускайте только в изолированном сегменте/VM и добровольно ограничивайте доступ (firewall-правило, разрешающее только нужные IP).
- Для доступа за пределами `localhost` добавьте базовую аутентификацию или поставьте перед приложением reverse proxy (nginx, Caddy) с авторизацией.
- Не выставляйте этот GUI в открытый интернет — он специально не предназначен для этого и является хорошей учебной целью именно как пример незащищённого dev-сервера.
