# MPDB (MITRE Parser Databases) GUI

Графический веб-интерфейс для управления парсером баз данных MITRE (CAPEC, CWE, CVE, ATT&CK).

## 📋 Возможности

- **Панель управления**: Запуск и мониторинг процессов парсинга, связывания, автозаполнения и перевода (AI-обогащение — в разработке), скачивание PDF-отчёта и переход в локальную папку `output/`
- **Редактор баз данных**: Просмотр, поиск и редактирование записей во всех 4 базах данных
- **Связи**: Граф перекрёстных связей записи (глубина 1-3 шага) с деталями записи по клику средней кнопкой мыши и переходом в редактор, а также поиск пути между двумя записями — см. [docs/features/link-graph.md](../docs/features/link-graph.md)
- **Редактор кэша переводов**: Управление кэшем переводов, редактирование и удаление записей
- **Настройки**: Конфигурация параметров перевода и AI

## 🚀 Установка

1. Установите зависимости:
```bash
pip install -r gui/requirements.txt
```

2. Запустите GUI:
```bash
python gui/run_gui.py
```

3. Откройте браузер и перейдите по адресу: http://localhost:5000

## 🎯 Использование

### Панель управления

- **Парсинг**: Скачивание и обработка исходных данных из источников MITRE
- **Связывание**: Установление перекрестных ссылок между базами данных
- **AI обогащение**: Заполнение пустых полей с помощью искусственного интеллекта
- **Перевод**: Перевод выбранных полей на русский язык

### Редактор баз данных

1. Выберите базу данных из меню "Базы данных"
2. Используйте поиск для фильтрации записей
3. Нажмите кнопку редактирования (карандаш) для изменения записи
4. Нажмите "Сохранить" для сохранения изменений

### Связи

1. **Граф связей записи** — найдите запись по id или названию, выберите глубину графа (1-3 шага). Клик средней кнопкой мыши по узлу показывает краткое описание под графом с кнопкой "В базе данных" для перехода в редактор
2. **Путь между записями** — введите записи в поля "Откуда" и "Куда" (с автодополнением), чтобы найти цепочку перекрёстных связей между ними через CAPEC/CWE/CVE/ATT&CK

### Кэш переводов

- Просмотр всех переводов
- Редактирование отдельных переводов
- Удаление некачественных переводов
- Полная очистка кэша

### Настройки

- Выбор сервиса перевода (Google/Yandex)
- Настройка параметров перевода (размер батча, задержка, попытки)
- Конфигурация AI провайдера (Ollama/OpenAI)

## 🔧 Технологии

- **Backend**: Flask, Flask-SocketIO
- **Frontend**: Bootstrap 5, jQuery, Socket.IO
- **Таблицы**: DataTables

## ⚠️ Примечания

- GUI требует наличия установленных баз данных в папке `output/`
- Для корректной работы необходимо запустить парсинг данных перед использованием редактора
- Изменения в настройках вступают в силу после перезапуска приложения
