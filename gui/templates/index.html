Что такое MITRE?

MITRE Corporation — американская некоммерческая организация, которая управляет несколькими ключевыми базами данных в области кибербезопасности. Эти базы используются по всему миру для классификации уязвимостей, шаблонов атак и поведения злоумышленников.

Ваш парсер объединяет четыре основные базы MITRE, обеспечивая перекрёстные ссылки и обогащение данных. Ниже представлены детали каждой базы.

CAPEC
Common Attack Pattern Enumeration and Classification

Каталог шаблонов атак — описывает, как злоумышленники осуществляют атаки на программное обеспечение.

  • Цель: Помочь разработчикам понять и предотвратить типовые атаки.
  • Примеры: SQL-инъекции, обход аутентификации, сниффинг.
  • Связано с: CWE (причины уязвимостей).
CWE
Common Weakness Enumeration

Список типов уязвимостей программного обеспечения.

  • Цель: Стандартизировать описание слабых мест в коде.
  • Примеры: Переполнение буфера, неверная проверка ввода.
  • Связано с: CAPEC (как атака использует слабость) и CVE (конкретные экземпляры).
CVE
Common Vulnerabilities and Exposures

Словарь известных уязвимостей в конкретных продуктах.

  • Цель: Уникальные идентификаторы для каждой известной уязвимости.
  • Пример: CVE-2024-12345 — уязвимость в библиотеке X.
  • Связано с: CWE (тип уязвимости) и CAPEC (способ атаки).
MITRE ATT&CK
Adversarial Tactics, Techniques & Common Knowledge

База тактик, техник и процедур, используемых злоумышленниками на всех этапах атаки.

  • Цель: Моделирование поведения атакующих.
  • Тактики: Разведка, закрепление, кража данных и др.
  • Связано с: CVE (уязвимости могут соответствовать техникам ATT&CK).
Как базы связаны между собой?

Ваш парсер автоматически строит перекрёстные ссылки, чтобы вы могли видеть полную картину:

  • CWE → CAPEC: какие атаки могут использовать данную уязвимость.
  • CVE → CWE: к какому типу относится конкретная уязвимость.
  • CVE → MITRE ATT&CK: какие техники атак связаны с уязвимостью.
  • CAPEC → CWE: какие слабости эксплуатирует атака.

Используйте процессы «Связывание» и «Автозаполнение» на панели управления, чтобы получить максимум информации.

Путь между записями

Найдите цепочку перекрёстных связей между двумя записями — например, между конкретной CVE и техникой ATT&CK, чтобы понять, через какие CWE и CAPEC они связаны.

Зачем нужна эта функция: базы CAPEC, CWE, CVE и ATT&CK редко связаны напрямую — обычно между ними цепочка из нескольких шагов (например, CVE → CWE → CAPEC → ATT&CK). Поиск пути строит эту цепочку автоматически, что удобно для анализа угроз: по уязвимости понять, какими шаблонами атак и техниками противника она может быть использована, и наоборот.

Как пользоваться:

  1. Введите id или название записи в поле «Откуда» (например, CVE-2021-44228, CAPEC-66, CWE-89 или T1059) и выберите вариант из выпадающего списка.
  2. То же самое для поля «Куда».
  3. Нажмите кнопку поиска — ниже появится цепочка записей и связей между ними.

Если путь не найден: поиск ограничен 6 шагами, а часть записей (особенно техники ATT&CK и некоторые CAPEC/CWE) в данных MITRE вообще не имеет перекрёстных связей с другими базами — для них путь в принципе не существует, это сообщение покажет явно.
