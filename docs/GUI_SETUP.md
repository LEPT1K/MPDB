# 🖥️ MPDB — Настройка и запуск графического интерфейса (GUI)

## 📋 Требования

- Python 3.8+
- Установленные зависимости проекта (см. `requirements.txt`)

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

(зависимости GUI также продублированы в `gui/requirements.txt`)

### 2. Запуск GUI

```bash
python gui/run_gui.py
```

### 3. Открыть в браузере

Перейдите по адресу: **http://localhost:5000** (или 5001, если 5000 занят)

## 🎯 Возможности GUI

### Панель управления
- **Парсинг** - скачивание и обработка баз данных MITRE (CAPEC, CWE, CVE, ATT&CK)
- **Связывание** - установление перекрестных ссылок между базами
- **Автозаполнение** - заполнение пустых полей (связи, факты из описаний, detection из STIX, шаблонные фразы)
- **AI обогащение** - 🚧 находится в разработке (запуск временно недоступен)
- **Перевод** - многопоточный перевод выбранных полей на русский язык
- Каждый процесс можно остановить кнопкой «Стоп»
- Карточки «Отчёты» (скачать PDF) и «Локальные файлы» (открыть папку `output/` с JSON-базами на этом компьютере)

### Редактор баз данных
- Просмотр всех 4 баз данных
- Поиск и фильтрация записей
- Редактирование полей
- Пагинация

### Связи
- **Граф связей записи** - интерактивная визуализация перекрёстных ссылок с переключением глубины (1-3 шага)
- Клик средней кнопкой мыши по узлу графа открывает карточку с описанием записи и кнопкой перехода в редактор базы данных
- **Путь между записями** - поиск цепочки связей между двумя записями через CAPEC/CWE/CVE/ATT&CK, с подсказкой по использованию
- Подробнее: [features/link-graph.md](features/link-graph.md)

### Кэш переводов
- Пагинация (25–250 записей на страницу)
- Поиск по оригиналу и переводу
- Длинные тексты свёрнуты до 3 строк — клик по ячейке разворачивает
- Редактирование и удаление записей, очистка кэша

### Настройки
- Лимиты парсинга (количество записей на базу)
- Конфигурация перевода: сервис (Google/Yandex), число потоков, задержка, число попыток
- AI настройки - 🚧 находятся в разработке, изменение параметров заблокировано

## 🔧 Интеграция с основным проектом

GUI автоматически:
- Использует конфигурацию из `src/config.py`
- Запускает скрипты из `src/` (step1_parse.py, step2_link.py, step4_autofill.py, translate_fields.py)
- Работает с данными из папки `output/`

## ⚠️ Важные замечания

1. **Первый запуск**: Перед использованием редактора баз данных необходимо выполнить парсинг (кнопка "Парсинг" на панели управления)

2. **Изменение настроек**: Настройки лимитов и перевода, сохранённые в GUI, записываются прямо в `src/config.py` и применяются при следующем запуске процессов

3. **Остановка GUI**: Нажмите `Ctrl+C` в терминале

4. **Порт**: Если порты 5000 и 5001 заняты, измените список портов в `gui/run_gui.py`:
   ```python
   ports = [5000, 5001]
   ```

5. **Безопасность**: GUI слушает все интерфейсы (`0.0.0.0`) и не имеет аутентификации — предназначен только для локального использования. Не открывайте порт 5000 в сеть/интернет. Подробнее: [../README.md#-безопасность-при-развёртывании](../README.md#-безопасность-при-развёртывании)

## 🆘 Решение проблем

### GUI не запускается
Проверьте, установлены ли зависимости:
```bash
pip install Flask Flask-SocketIO Flask-CORS python-socketio
```

### Ошибки при запуске процессов
Убедитесь, что основной проект настроен корректно:
```bash
python src/main.py
```

### Не отображаются базы данных
Проверьте наличие файлов в папке `output/`:
- capec_database.json
- cwe_database.json
- cve_database.json
- mitre_attack.json

Если файлов нет, запустите парсинг через GUI или консоль.

## 📞 Поддержка

При возникновении проблем:
1. Проверьте логи в консоли
2. Проверьте логи в GUI (панель "Логи процессов")
3. Убедитесь, что все зависимости установлены
