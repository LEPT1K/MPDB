# MPDB — Руководство пользователя

**MPDB** (MITRE Parser Databases) — инструмент для парсинга, обработки и анализа баз данных MITRE (CAPEC, CWE, CVE, ATT&CK) с веб-интерфейсом.

Это руководство описывает, как установить, запустить и использовать MPDB: от первого запуска до работы со всеми страницами графического интерфейса.

---

## 1. О проекте

MPDB скачивает и обрабатывает четыре базы данных MITRE:

| База | Расшифровка | Содержимое |
|---|---|---|
| **CAPEC** | Common Attack Pattern Enumeration and Classification | Шаблоны атак |
| **CWE** | Common Weakness Enumeration | Типы уязвимостей (слабости) ПО |
| **CVE** | Common Vulnerabilities and Exposures | Известные уязвимости (NVD) |
| **MITRE ATT&CK** | Adversarial Tactics, Techniques & Common Knowledge | Тактики и техники злоумышленников |

После обработки записи связываются между собой перекрёстными ссылками (например, CVE → CWE → CAPEC → ATT&CK), пустые поля автоматически заполняются, а текстовые поля переводятся на русский язык. Результат сохраняется в JSON-файлы в папке `output/` и доступен для просмотра, редактирования и экспорта через веб-интерфейс.

## 2. Установка

### Требования

- **Python 3.8+** (если запускаете из исходного кода)
- Подключение к интернету (для скачивания баз MITRE и перевода)
- Для готового `.exe` — Python не требуется

### Установка зависимостей (запуск из исходного кода)

```bash
pip install -r requirements.txt
```

Зависимости GUI продублированы в `gui/requirements.txt`:

```bash
pip install -r gui/requirements.txt
```

## 3. Способы запуска

### 3.1 Готовый `.exe` (рекомендуется для конечного пользователя)

В папке `dist/MPDB/` находится собранное приложение `MPDB.exe`. Достаточно запустить его двойным щелчком — откроется консольное окно с логом запуска, а в браузере автоматически откроется веб-интерфейс по адресу `http://localhost:5000` (или `5001`, если порт 5000 занят).

Для остановки закройте консольное окно или нажмите `Ctrl+C` в нём.

> Базы данных (`output/*.json`) находятся рядом с `.exe`, в папке `dist/MPDB/output/`. Настройки хранятся в `dist/MPDB/_internal/src/config.py`.

### 3.2 Графический интерфейс из исходного кода

```bash
python gui/run_gui.py
```

Откройте браузер: **http://localhost:5000**

### 3.3 Консольное меню

```bash
python src/menu.py
```

Меню предлагает выбрать этап обработки данных в текстовом виде (см. раздел 6).

### 3.4 Прямой запуск всех этапов подряд

```bash
python src/run_all.py
```

Выполняет парсинг → связывание → автозаполнение → перевод без интерактивного выбора.

## 4. Быстрый старт

Типичный порядок действий при первом использовании:

1. Запустите приложение (`MPDB.exe` или `python gui/run_gui.py`).
2. Откройте браузер на `http://localhost:5000`.
3. Перейдите на страницу **"Главная"**.
4. В блоке **"Управление процессами"** нажмите **"Запустить"** на карточке **"Парсинг"** — начнётся скачивание и разбор баз CAPEC/CWE/CVE/ATT&CK. Прогресс отображается в окне **"Логи процессов"**.
5. После завершения парсинга запустите **"Связывание"** — заполнятся перекрёстные ссылки между базами.
6. Запустите **"Автозаполнение"** — пустые поля будут заполнены на основе связей, эвристик и шаблонов.
7. При необходимости запустите **"Перевод"** — выбранные текстовые поля переводятся на русский.
8. Перейдите в раздел **"Базы данных"** для просмотра и редактирования результатов, либо в **"Связи"** для анализа перекрёстных ссылок.
9. На дэшборде доступна кнопка скачивания **PDF-отчёта** и кнопка открытия локальной папки `output/`.

> Каждый процесс можно прервать кнопкой **"Стоп"** на соответствующей карточке.

## 5. Графический интерфейс

### 5.1 Главная

Стартовая страница содержит:

- Краткое объяснение, что такое MITRE, CAPEC, CWE, CVE и ATT&CK, и как они связаны друг с другом.
- **Лента обновлений MITRE баз** — показывает, актуальны ли локальные базы по сравнению с первоисточниками MITRE/NVD (дата последнего обновления источника, статус "актуально"/"есть обновление"). Кнопка обновления ленты — иконка со стрелками в заголовке блока. Клик по источнику открывает подробности (объём данных, дата проверки, количество локальных записей).

### 5.2 Дэшборд

Главная рабочая страница:

- **Карточки статистики** — общее число баз, записей, переводов в кэше и перекрёстных связей.
- **Управление процессами** — пять карточек: "Парсинг", "Связывание", "Автозаполнение", "AI-обогащение" (🚧 в разработке, кнопки отключены) и "Перевод". У каждой карточки кнопки **"Запустить"** и **"Стоп"**.
- **Отчёты** — кнопка скачивания PDF-отчёта со сводной статистикой по всем базам и связям.
- **Локальные файлы** — кнопка открытия папки `output/` в проводнике/файловом менеджере прямо на компьютере, где запущено приложение.
- **Логи процессов** — окно реального времени с цветовой кодировкой событий (информация/предупреждение/ошибка/успех); кнопка разворачивания на весь экран и кнопка очистки.

### 5.3 Редактор баз данных

Открывается через выпадающее меню **"Базы данных"** в навигации: CAPEC, CWE, CVE или MITRE ATT&CK.

Возможности:

- **Поиск и фильтрация** записей по id, названию или содержимому полей.
- **Пагинация** результатов.
- **Редактирование** полей записи прямо в таблице с сохранением через кнопку **"Сохранить"**.
- **Обновление** — кнопка перезагрузки данных с диска.
- **Экспорт** — выпадающее меню с форматами **CSV** и **Excel (XLSX)**.

Описание полей каждой базы — см. раздел 7 и [README.md](../README.md#-описание-полей-json-файлов).

### 5.4 Связи

Страница для анализа перекрёстных связей между CAPEC, CWE, CVE и ATT&CK. Подробное описание — [docs/features/link-graph.md](features/link-graph.md). Кратко:

- **Обзор связей между базами** — общая статистика количества перекрёстных ссылок и часто цитируемые записи.
- **Граф связей записи** — найдите запись по id (например, `CAPEC-66`, `CWE-89`, `CVE-2021-44228`, `T1059`) или названию, выберите глубину обхода (1–3 шага). Узлы окрашены по базе: CAPEC — синий, CWE — жёлтый, CVE — зелёный, ATT&CK — красный. Клик средней кнопкой мыши по узлу открывает карточку с описанием записи и кнопкой перехода в редактор соответствующей базы.
- **Путь между записями** — найдите кратчайшую цепочку связей между двумя записями (поля "Откуда" и "Куда" с автодополнением). Поиск ограничен 6 шагами; для некоторых записей путь может не существовать.

### 5.5 Кэш переводов

Страница для просмотра и редактирования кэша переводов (`output/translate_cache.json`):

- **Пагинация** (25–250 записей на страницу).
- **Поиск** по оригинальному тексту и переводу.
- Длинные тексты свёрнуты до 3 строк — клик по ячейке разворачивает полностью.
- **Редактирование и удаление** отдельных записей.
- Кнопка **"Очистить кэш"** — полностью удаляет кэш переводов (после этого все переводы будут выполнены заново).

### 5.6 Настройки

Конфигурация приложения (записывается прямо в `src/config.py`):

- **Лимиты парсинга** — максимальное количество записей для каждой из 4 баз (0 = без ограничений).
- **Перевод** — выбор сервиса (Google/Yandex), число параллельных потоков, задержка между запросами, число повторных попыток при ошибке.
- **AI** — 🚧 находится в разработке, изменение параметров временно заблокировано в интерфейсе.

## 6. Этапы обработки данных

### 6.1 Парсинг

Скачивает актуальные данные из официальных источников MITRE/NVD, разбирает XML/JSON и сохраняет в `output/*.json`. Лимиты по количеству записей задаются в настройках (`MAX_*_RECORDS`).

### 6.2 Связывание (Cross-Linking)

Анализирует перекрёстные ссылки между загруженными базами:

- Заполняет `related_capec`, `related_cwe`, `related_mitre` там, где они отсутствуют, на основе обратных и цепочечных связей.
- Переносит меры защиты (`mitigations`/`mitigation`) между связанными записями (CVE ← CWE ← CAPEC ← ATT&CK).

### 6.3 Автозаполнение (Autofill)

Ярусная стратегия — от точного к шаблонному:

- **Ярус A**: восстановление связей по реальным ID (реверс и транзитивность, например ATT&CK → CAPEC → CWE).
- **Ярус B**: извлечение фактов из описаний — порты, сервисы, тип атаки, категория CWE, severity из CVSS.
- **Ярус C**: `detection` для ATT&CK из STIX-первоисточника (детекция → аналитика) с переводом. Требует доступа к GitHub (~50 МБ) — можно указать локальный файл (`--stix путь`) или пропустить (`--no-detection`).
- **Ярус D**: шаблонные фразы (настраиваются в `Config.AUTOFILL_TEMPLATES`) для оставшихся пустых текстовых полей. Поля-связи (`related_*`) шаблонами не заполняются — только реальные ID.

### 6.4 Перевод (Translation)

Многопоточный перевод выбранных текстовых полей на русский язык (по умолчанию 15 потоков). Используется глобальная дедупликация и постоянный кэш (`output/translate_cache.json`) — повторные запуски почти мгновенны. Поддерживаются Yandex Translate и Google Translate.

## 8. Конфигурация (`src/config.py`)

Основные параметры:

```python
# Перевод
ENABLE_TRANSLATION = False          # Включить перевод при парсинге
TRANSLATE_TO = "ru"                 # Целевой язык
TRANSLATION_DELAY = 0.15            # Задержка перед каждым запросом в потоке (сек)
TRANSLATION_WORKERS = 15            # Число параллельных потоков перевода
TRANSLATION_MAX_RETRIES = 5         # Число повторных попыток при ошибке
TRANSLATION_SERVICE = "google"      # google или yandex

# Автозаполнение — шаблонные фразы для пустых полей
AUTOFILL_TEMPLATES = { ... }         # см. config.py

# AI (в разработке)
AI_PROVIDER = "ollama"              # ollama или openai
AI_MODEL = "qwen2.5:3b"             # Модель для Ollama
AI_BASE_URL = "http://localhost:11434/v1"

# Ограничения (0 = все записи)
MAX_CAPEC_RECORDS = 1000
MAX_CWE_RECORDS = 1000
MAX_CVE_RECORDS = 4000
MAX_ATTACK_RECORDS = 1000
```

Эти же параметры (кроме AI) доступны на странице **"Настройки"** в GUI.

## 9. Решение проблем

### GUI не запускается

```bash
pip install -r gui/requirements.txt
```

### Порт 5000 занят

Приложение автоматически попробует порт 5001. Если оба заняты — закройте другие приложения или измените список портов в `gui/run_gui.py` (`ports = [5000, 5001]`).

### Ошибки при парсинге

- Проверьте подключение к интернету.
- Увеличьте таймауты в `config.py`.
- Попробуйте запустить с меньшим количеством записей (лимиты `MAX_*_RECORDS`).

### Проблемы с переводом

- Проверьте API-ключ для Yandex (если используется).
- При ошибках 429 (Too Many Requests) уменьшите `TRANSLATION_WORKERS` или увеличьте `TRANSLATION_DELAY`.

### Автозаполнение не скачивает STIX

Ярус C требует доступ к GitHub (~50 МБ). Можно указать локальный файл:

```bash
python src/step4_autofill.py --stix путь/к/enterprise-attack.json
```

либо пропустить ярус:

```bash
python src/step4_autofill.py --no-detection
```

### Не отображаются базы данных в редакторе

Проверьте наличие файлов в папке `output/`: `capec_database.json`, `cwe_database.json`, `cve_database.json`, `mitre_attack.json`. Если их нет — запустите парсинг через дэшборд или консоль.

## 10. Безопасность при развёртывании

GUI предназначен **только для локального использования** (`localhost`):

- Сервер слушает все интерфейсы (`0.0.0.0:5000`) — доступен любому устройству в той же сети, а при проброшенном порте и из интернета.
- Используется dev-сервер Werkzeug без защиты от нагрузки и production-механизмов.
- **Аутентификация отсутствует** — любой, кто достучится до порта, может запускать/останавливать процессы парсинга и перевода, а также записывать произвольный JSON в `output/` через API редактора баз данных.

Рекомендации:

- Используйте `http://localhost:5000` и не открывайте порт 5000 наружу (роутер/файрвол).
- Для доступа из локальной сети — только изолированный сегмент/VM с ограничением по IP.
- Для доступа за пределами `localhost` — добавьте аутентификацию или reverse proxy (nginx, Caddy) с авторизацией.
- Не выставляйте GUI в открытый интернет.

Подробнее: [README.md — раздел "🔒 Безопасность при развёртывании"](../README.md#-безопасность-при-развёртывании)
